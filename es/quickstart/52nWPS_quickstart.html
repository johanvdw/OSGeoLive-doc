Guía de Inicio Rápido 52 North WPS¶

El 52 ° North WPS es un marco basado en la web de procesamiento de datos (espaciales). El servicio implementa el estándar de interfaz OGC Web Servicio de Procesamiento 1.0.

This quickstart describes how to process data and store the result in another web service using WPS operations.

Ejecución¶

  • Para ejecutar 52°North WPS en el Live DVD, vaya a Geospatial ‣ Web Services ‣ 52North ‣ Start 52North WPS para iniciar el 52°North WPS o utilice este enlace directo. (Si no puede conectarse con el servicio, intente inicar el motor Tomcat servlet siguiendo los pasos al pie de SOS quickstart page.)

  • Aparecerá la página de bienvenida de 52n WPS.

Obtener información sobre el servicio¶

  • Consulte las capacidades de 52 ° North WPS con esta solicitud:

    http://localhost:8080/52nWPS/WebProcessingService?Request=GetCapabilities&Service=WPS

Datos de proceso¶

  • Click en el enlace para abrir el 52nWPS-TestClient.

  • Asegúrese de que GeoServer se está ejecutando, ya que la demostración requiere datos de la instalación local de GeoServer. Para ello, intente abrir la página http://localhost:8082/geoserver/web. Si no se pudo encontrar la página, seleccione Geospatial ‣ Web Services ‣ GeoServer ‣ Start GeoServer en el menú. El GeoServer debe ejecutarse después de unos momentos.

  • Seleccione la solicitud de ejemplo «SimpleBuffer_out_wfs.xml» de la lista desplegable:

  • Presione el Botón de Enviar y la solicitud será transmitida hacia el 52°North WPS, el cual generará buffer alrededor de las carreteras principales de Tasmania con un ancho de 0.05 grados y almacenará el resultado en GeoServer.

  • Copie el ResourceID de la respuesta. Ese es el nombre de la capa de GeoServer. Agregue el ID a la siguiente solicictud: http://localhost:8082/geoserver/wms?service=WMS&version=1.1.0&request=GetMap&styles=&bbox=145.14757902405984,-43.47330297262748,148.32274986232298,-40.80286290459129&width=512&height=430&srs=EPSG:4326&format=application/openlayers&layers=Add-ResourceID-here Deberá obtener el siguiente resultado.

Cosas que probar¶

  • Pruebe otros ejemplos de solicitudes de la lista desplegable del cliente de prueba WPS.
  • Usted puede también instalar el complemento WPS para QGIS y pruébalo con el WPS de 52°North.

¿Qué sigue?¶

  • Cree sus propios procesos y ejecútelos usando varios clientes.
  • Configurar el WPS para ejecutar los procesos de GRASS GIS 7.
  • Configurar el WPS para ejecutar los scripts de R como procesos.

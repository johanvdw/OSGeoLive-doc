Guide démarrage rapide de l’OSGeoLive pour l’exécution dans une machine virtuelle¶

Ce guide décrit une manière d’exécuter l’OSGeoLive dans une machine virtuelle VirtualBox. Pour les autres méthodes, suivez les liens depuis la section « Voir aussi » plus bas.

Avantages des machines virtuelles¶

  • La vitesse de réponse d’un VM est beaucoup plus rapide que sur le DVD, et généralement comparable à une installation directement sur la machine hôte.
  • Tous les changements de configuration, les mises à jour logicielles et les fichiers enregistrés sont permanents, comme sur n’importe quel ordinateur « normal ». Ainsi:
  • Vous pouvez personnaliser et mettre à jour le système
  • Vous pouvez sauvegarder votre travail dans la VM
  • Vous pouvez installer des logiciels supplémentaires
  • Vous pouvez sauvegarder l’installation de votre machine virtuelle.

Prérequis système¶

  • RAM: 2 Go minimum. Le système Lubuntu fonctionne bien avec 1 Go de RAM, et vous devrez conserver au moins la même quantité de mémoire pour votre système hôte. Un total de 2 Go sera donc nécessaire pour un fonctionnement en douceur. Certaines applications, comme GeoServer, nécessitent que la machine virtuelle dispose d’au moins 2 Go de RAM. Donc, si possible, pensez à fournir 2 Go ou plus pour votre machine virtuelle.
  • Espace disque dur disponible: le fichier de disque virtuel (vmdk) de: live.osgeo.org décompressé fait presque 10 GB. Et vous voudrez sûrement plus d’espace pour pouvoir travailler sur la machine virtuelle. Ainsi 20 Go est une bonne recommandation.
  • Processeur: Tout CPU fera l’affaire, mais un processeur qui supporte la « Technologie de virtualisation » sera plus rapide. Vous pouvez vérifier le CPU de votre ordinateur ici.

La plupart des machines produites au cours des 4-5 dernières années répondront à ces besoins.

Howto¶

Téléchargements

Téléchargez Virtual Box pour votre plateforme et installez le logiciel. Sur Windows, exécutez l’installateur, ou sur les systèmes linux similaires à Ubuntu, faites ce qui suit:

apt-get install virtualbox-ose

En plus, téléchargez fichier de disque virtuel OSGeo (vmdk) depuis live.osgeo.org . Cliquez sur le bouton « Téléchargez la machine virtuelle ». Ensuite, décompresser le fichier téléchargé (en utilisant 7zip ).

Créer une machine virtuelle

Démarrez l’application VirtualBox et cliquez sur le bouton « New » pour créer une nouvelle VM, puis sur « Next ».

Entrez un nom tel que OSGeoLive, et choisissez Linux comme « Operating system », et Ubuntu comme « Version ».

Dans l’écran suivant, réglez la mémoire à 1024 Mo (ou plus si votre ordinateur hôte a plus de 4 Go, comme sur la capture d’écran).

Passez à l’écran suivant et choisissez « Use existing hard disk ». Cliquez maintenant sur le bouton (une icône de dossier) pour naviguer jusqu” à l’endroit où vous avez sauvegardé le fichier vmdk OSGeoLive. Sélectionnez ce fichier, puis appuyez sur Next et sur Create.

Astuces de configuration et réglages

Une fois la VM créée, cliquez sur le bouton Paramètres. Dans la section « General », allez dans l’onglet « Advanced » et cliquez sur « Show at top of screen » pour avoir la barre d’outils Mini.

Go to the « Display » section and increase video memory to 32 or 64 MB. In some host operating systems, it may be required to « Enable 3D Acceleration » for display to work properly.

En plus, passez à la section « Répertoires partagés » et cliquez sur « Ajouter un dossier » (icône + verte à droite) pour trouver un répertoire sur votre ordinateur hôte que vous souhaitez partager dans la VM.

Vous pouvez sélectionner de rendre le répertoire partagé en lecture seule et de le monter automatiquement. Une fois le « Chemin du dossier » et le « Nom du dossier » définis, cliquez sur OK, puis sur OK de nouveau pour terminer et fermer la fenêtre des paramètres.

Exécuter une machine virtuelle

Démarrez maintenant la VM en cliquant sur le bouton Démarrer (flèche verte).

Une fois que le système OSGeo apparaît, ajoutez-vous au groupe vboxsf pour que les répertoires partagés (définis ci-dessus) soient accessibles en exécutant dans une fenêtre de terminal:

user@osgeolive:~$ sudo usermod -a -G vboxsf user

Dans l’exemple ci-dessus, nous avons défini un chemin de répertoire partagé sur le système hôte et l’avons nommé « GIS » dans les paramètres de la VM. Le répertoire partagé apparaîtra dans le système de fichiers sous /media/sf_GIS/. Pour monter ce dossier dans le répertoire personnel de l’utilisateur, par exemple, dans un terminal, faites:

user@osgeolive:~$ mkdir GIS

user@osgeolive:~$ sudo mount -t vboxsf -o uid=user,rw GIS /home/user/GIS

Maintenant, les fichiers sur votre machine hôte dans le dossier choisi seront également disponibles dans la VM dans le dossier « GIS » du répertoire personnel de l’utilisateur.
